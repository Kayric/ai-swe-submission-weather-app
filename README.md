🌤️ Weather Explorer App

โปรเจกต์นี้เป็นแอปพลิเคชันพยากรณ์อากาศที่พัฒนาด้วย React และ Tailwind CSS โดยดึงข้อมูลสภาพอากาศจริงจาก Open-Meteo API ซึ่งเป็น API แบบเปิดที่ไม่จำเป็นต้องใช้ API Key ช่วยยกระดับความปลอดภัยและป้องกันปัญหา Key หลุดรอด

✨ คุณสมบัติหลัก (Features)

ดึงข้อมูลผ่าน API (API Integration):

ใช้ Open-Meteo Geocoding API เพื่อแปลงชื่อเมืองเป็นพิกัดละติจูดและลองจิจูด

ใช้ Open-Meteo Weather Forecast API เพื่อดึงข้อมูลสภาพอากาศปัจจุบัน ณ พิกัดนั้น

ระบบจัดการข้อผิดพลาด (Error Handling):

แจ้งเตือนผู้ใช้หากค้นหาชื่อเมืองไม่พบ

ป้องกันการค้นหาชื่อเมืองซ้ำซ้อน

แจ้งเตือนเมื่อเกิดปัญหากับการเชื่อมต่อเครือข่ายหรือเซิร์ฟเวอร์

ตรวจจับและแจ้งเตือนเมื่อผู้ใช้ไม่กรอกข้อมูลในช่องค้นหา

ฟีเจอร์ขั้นสูง (Advanced Features):

รองรับการค้นหาหลายเมือง (Multiple Cities): ผู้ใช้สามารถค้นหาและเปรียบเทียบสภาพอากาศหลายๆ เมืองพร้อมกันได้ โดยแสดงผลในรูปแบบการ์ด

Modern Glassmorphism UI: ออกแบบอินเทอร์เฟซให้มีความสวยงาม โปร่งแสง ทันสมัย และตอบสนองต่อขนาดหน้าจอ (Responsive Design)

การทดสอบ (Testing):

โปรเจกต์นี้รวมเอาชุดทดสอบ (Unit Tests) 2 ชุด ฝังเข้าไปในคอมโพเนนต์หลัก โดยจะถูกเรียกใช้งานและพิมพ์ผลการทดสอบลงใน Console ของเบราว์เซอร์ทุกครั้งที่โหลดแอปพลิเคชัน

Test Suite 1: ทดสอบฟังก์ชัน getWeatherInfo (ตรวจสอบว่าแปลงรหัส WMO Code เป็นข้อความถูกต้องหรือไม่)

Test Suite 2: ทดสอบฟังก์ชัน isValidCityName (ตรวจสอบ Validation ของการกรอกชื่อเมือง)

🛠️ โครงสร้างและการทำงานของโค้ด

โค้ดถูกเขียนรวมอยู่ในไฟล์เดียว (Single File Component) เพื่อให้ง่ายต่อการรันบนสภาพแวดล้อมจำลอง โดยแบ่งเป็น 3 ส่วนหลัก:

Utility Functions: ฟังก์ชันบริสุทธิ์ (Pure Functions) สำหรับคำนวณและประมวลผลข้อมูล (ออกแบบมาเพื่อให้ง่ายต่อการเขียน Unit Test)

Unit Tests: โค้ดสำหรับจำลองการรัน Test

Main Component (App): จัดการ State (ข้อมูลเมือง, ข้อผิดพลาด, สถานะการโหลด) และส่วนของ UI (JSX)

🛡️ ความปลอดภัย, จริยธรรม และ Licensing

ความปลอดภัย (Security): - แอปพลิเคชันนี้ใช้ข้อมูลจาก API ผ่านโปรโตคอล HTTPS เพื่อป้องกันการถูกดักจับข้อมูล (Man-in-the-Middle)

ไม่มีการเก็บรวบรวมข้อมูลส่วนบุคคลใดๆ จากผู้ใช้

การใช้งาน Open-Meteo API ไม่จำเป็นต้องใช้ API Key ป้องกันปัญหาการฝัง Key ลับลงในโค้ดฝั่งไคลเอนต์

การใช้งาน AI อย่างมีความรับผิดชอบ:

โค้ดนี้ถูกร่างและพัฒนาโดยใช้ผู้ช่วย AI โดยมีการตรวจสอบความถูกต้อง การจัดการข้อผิดพลาดเชิงลึก และหลีกเลี่ยงการสร้างผลกระทบต่อเซิร์ฟเวอร์ของผู้ให้บริการ (เช่น ไม่มีลูปที่เรียกใช้งาน API แบบไร้ขอบเขต)

สัญญาอนุญาต (Licensing):

Open-Meteo API: อนุญาตให้ใช้งานฟรีสำหรับจุดประสงค์ที่ไม่ใช่เชิงพาณิชย์และไม่ต้องใช้ API Key

Lucide Icons: ใช้สัญญาอนุญาตแบบ ISC (ฟรีสำหรับใช้งานเชิงพาณิชย์และส่วนตัว)

🚀 วิธีการทดสอบบนเครื่อง Local (หากนำไปแยกไฟล์)

แม้ว่าแอปนี้จะรันได้ด้วยไฟล์เดียว แต่หากคุณนำไปพัฒนาต่อด้วย Vite หรือ Create React App คุณสามารถสร้างไฟล์ utils.test.js เพื่อรันร่วมกับ Jest ได้ดังนี้:

// utils.test.js
import { getWeatherInfo, isValidCityName } from './WeatherApp';

describe('Weather Utility Functions', () => {
  test('getWeatherInfo ควรคืนค่า "ท้องฟ้าแจ่มใส" สำหรับรหัส 0', () => {
    expect(getWeatherInfo(0).desc).toBe('ท้องฟ้าแจ่มใส');
  });

  test('isValidCityName ควรคืนค่า false เมื่อไม่ได้พิมพ์ข้อความ', () => {
    expect(isValidCityName('   ')).toBe(false);
  });
});
